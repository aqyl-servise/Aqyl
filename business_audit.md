## 2. AI-функции

Платформа использует **Anthropic Claude** через официальный SDK (`@anthropic-ai/sdk ^0.90.0`).

### Модели

| Модель | ID | Применение |
|---|---|---|
| Claude Haiku | `claude-haiku-4-5-20251001` | Быстрые/частые операции (чат, задания, аналитика) |
| Claude Sonnet | `claude-sonnet-4-20250514` | Сложные генерации (КМЖ этапы, презентации) |

### AI-функции по модулям

| Модуль | Что генерирует | Модель | Max Tokens | Доступ |
|---|---|---|---|---|
| **KMZh** (цели урока) | Цели "все/большинство/некоторые" для 3 уровней | Haiku | 300 | teacher, class_teacher |
| **KMZh** (этапы урока) | Полная структура урока (5–7 этапов с действиями учителя и ученика) | Sonnet | 2000 | teacher, class_teacher |
| **AI Chat** | Ответы на вопросы учителя в контексте раздела | Haiku | 400 | teacher, class_teacher |
| **Generators** (задания) | Набор заданий по предмету/теме/классу | Haiku | 800 | teacher, class_teacher |
| **Generators** (lesson plan) | Поурочный план | Haiku | ~800 | teacher, class_teacher |
| **FL** | Задачи по функциональной грамотности (тест/открытый/практический) | Haiku | 800 | teacher, class_teacher |
| **Materials** (презентации) | JSON структура слайдов → реальный PPTX | Sonnet | 2000 | teacher, class_teacher |
| **Materials** (иллюстрации) | SVG-код образовательной иллюстрации | Sonnet | ~1500 | teacher, class_teacher |
| **Analytics** | Анализ успеваемости школы на языке директора | Haiku | 600 | admin, principal |
| **Questionnaires** | Генерация анкеты по теме/классу/типу вопросов | Haiku | ~800 | admin, principal |

### Управление токенами и стоимостью

- `TokenService` — дедактирует токены за каждый AI-запрос, хранит в БД (`TokenPackage`, `TokenUsage`)
- `AiUsageService` — дневные лимиты по пользователю (20 запросов/день по умолчанию)
- `AiUsagePanelAdmin` — дашборд для директора: расходы в тенге, самый активный учитель, график по дням
- **KMZh-кэш**: идентичные запросы (предмет + тема + класс) отдаются из БД без вызова API. Hit rate отображается в /kmzh-cache/stats

### Оценка токенопотребления на типичный KMZh-план

```
Objectives (Haiku):   ~200 input + 80 output  = ~280 токенов
Stages (Sonnet):      ~800 input + 1200 output = ~2000 токенов
Итого на один план:   ~2280 токенов
При кэш-хите Stages:  ~280 токенов (экономия ~87%)
```

### 4.1 Реальные заглушки и ограниченный функционал

| Компонент | Проблема | Критичность |

### 4.4 Архитектурные риски

- **Хранение файлов на диске** (`./uploads/`): при горизонтальном масштабировании или миграции сервера все загруженные файлы нужно переносить вручную. Нет интеграции с S3/Cloudflare R2.
- **KMZH-кэш в PostgreSQL**: при большом числе уникальных запросов таблица кэша будет расти неограниченно. Нет TTL или ротации.
- **Отсутствие тестов**: файлы `*.spec.ts` отсутствуют в проверенных модулях — нет автоматизированного тестового покрытия.

**Оценка расходов на AI для одной школы (50 учителей):**
- 50 учителей × 2 KMZh/неделю × ~2000 токенов = 200K токенов/нед
- При кэш-хите 50%: ~100K Sonnet + 100K Haiku ≈ $0.40–0.80/нед → **~$2–4/мес на школу**

Платформа готова к production для основных ролей (учитель, директор, завуч, студент). Специализированные роли (психолог, соц.педагог, завуч по воспитанию) получают ограниченный функционал через файловые менеджеры и требуют доработки для полноценного использования.
